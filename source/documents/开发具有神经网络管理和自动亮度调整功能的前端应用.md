# 开发具有神经网络管理和自动亮度调整功能的前端应用

## 1. 项目结构设计

### 1.1 核心文件

- `app.py`：主应用程序，包含GUI界面、系统托盘功能和应用逻辑
- `liangdu.py`：包含亮度处理的核心功能、摄像头捕获和神经网络模型
- `config.py`：配置管理，存储应用设置
- `utils.py`：系统工具函数，如开机自启动设置、系统操作等
- `notification.py`：通知系统，用于显示桌面通知

### 1.2 资源文件

- `brightness_model.npz`：预训练的神经网络模型文件
- `favicon.ico`：应用图标

### 1.3 打包配置

- `app.spec`：PyInstaller打包配置文件
- `brightness_manager.spec`：主应用打包配置文件

### 1.4 依赖文件

- `requirements.txt`：项目依赖库列表

## 2. 功能模块实现

### 2.1 神经网络管理模块

- 实现模型自动加载与创建机制
- 支持多特征输入（亮度、对比度、亮度变化率、移动平均亮度）
- 实现模型预训练机制，使用随机生成的训练数据
- 实现模型保存与加载功能
- 支持ReLU和tanh激活函数
- 实现Adam优化器支持
- 实现缓存机制，提高推理效率

### 2.2 系统托盘模块

- 实现应用启动后自动最小化到系统托盘
- 实现左键点击打开管理界面
- 实现右键菜单，包含打开管理界面、自动调整亮度开关、立即调整亮度和退出选项
- 支持图标自定义

### 2.3 管理界面模块

- 设计信息展示文本框，实时显示系统状态和日志
- 实现屏幕亮度滑动条，支持手动调整亮度
- 实现应用按钮，用于手动应用亮度设置
- 实现训练按钮，用于训练神经网络模型
- 实现开机自启动勾选框，支持设置开机自启动

### 2.4 亮度处理模块

- 实现摄像头捕获功能，支持捕获多帧图像
- 实现亮度计算算法，使用加权平均亮度、标准差、亮度变化率等多个指标
- 实现特征提取，包括亮度、对比度、亮度变化率和移动平均亮度
- 实现数据缓存机制，限制缓存大小
- 实现神经网络预测亮度调整结果
- 实现根据预测结果自动调整滑动条位置

### 2.5 自动亮度调整模块

- 实现定时自动调整亮度功能（每5分钟执行一次）
- 实现超时机制，避免长时间阻塞
- 实现全面的错误处理机制
- 支持自动调整亮度开关控制

### 2.6 通知系统

- 实现桌面通知功能，支持显示各种系统状态
- 支持在主线程安全显示通知

### 2.7 系统集成模块

- 实现开机自启动设置功能
- 实现单实例运行检查，避免多个应用实例同时运行
- 实现Windows系统亮度调整的多种方法（WMI、PowerShell等）
- 支持管理员权限检查和提示

## 3. 技术选型

### 3.1 GUI库

- 使用 `tkinter`：Python标准库，无需额外安装，适合快速开发
- 使用 `ttk`：tkinter的主题扩展，提供更好的UI组件

### 3.2 系统托盘

- 使用 `pystray`：轻量级库，支持跨平台系统托盘功能

### 3.3 图像处理

- 使用 `OpenCV (cv2)`：用于摄像头捕获和图像处理
- 使用 `PIL/Pillow`：用于图像处理和图标加载

### 3.4 神经网络

- 使用 `numpy`：用于神经网络计算和数值处理

### 3.5 系统操作

- 使用 `ctypes`：用于Windows系统调用
- 使用 `winreg`：用于开机自启动设置
- 使用 `subprocess`：用于执行PowerShell命令调整亮度
- 使用 `win32api`、`win32con`、`win32event`、`win32gui`：用于Windows系统操作和单实例检查

### 3.6 通知系统

- 使用自定义 `notification` 模块：基于tkinter的通知系统

### 3.7 打包工具

- 使用 `PyInstaller`：用于将Python应用打包为可执行文件
